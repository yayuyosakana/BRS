# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## このディレクトリの性質

コードベースではなく、**医学部「行動科学」ユニットの学習教材**を集約した作業ディレクトリ。ユーザーは **グループ20** に所属し、グループ担当回のスライド作成と、自分用の予習・復習ノートを管理している。教科書は **BRS Behavioral Science 第8版**（ルートの `BRS Behavioral Science (8e).pdf`）が唯一の出典。

## ディレクトリ構成

- ルート直下の `M:D` 形式フォルダ（例：`4:28`, `5:26-3`, `6:9`）が **講義1回分** に対応。`5:26-3` `5:26-4` のように同日複数コマは枝番で区別
- `5:18` `5:25` `6:3` のような空フォルダは予定枠（資料未配布）
- `_変更前_Ch17_心理療法` は履修順序変更前の旧資料の保管
- `レビューテスト　第●回　グループ●2024.pptx` は提出スライドの**強制テンプレート**（4:3固定、改変不可）
- `レビューテスト担当表2026.pdf` は各グループの担当回が定義された一次資料

## 各講義フォルダのファイル種別と役割

| ファイル | 役割 | 編集権限 |
|---------|------|---------|
| `課題シート ChXX-….docx` | 教員配布の **一次資料**。「学習項目」「キーワード」「概要」を含む | **読むだけ・改変しない** |
| `予習ノート_ChXX_….md` | ユーザー個人の学習ノート。各章の体系的まとめ＋末尾に「**学習項目への回答**」セクション | 編集対象 |
| `レビューテスト_ChXX_<テーマ>.md` | BRS章末Review Testの全Q演習集。**1ファイル＝1章**、Q番号昇順、BRSの全Qを網羅 | 編集対象 |
| `提出用_スライド原稿_グループ20.md` | グループ担当回のスライド本文を **そのまま貼れる形** で記述。問題スライド＋解答解説スライドの2枚構成 | 編集対象 |
| `LINEメッセージ_グループ20.txt` | グループLINEへ送る依頼文の下書き | 編集対象 |

## 確立された規約（過去の指示で固定済み）

### 予習ノートの「学習項目への回答」セクション
- 課題シートの `学習項目（カッコ内は口頭説明の目安時間）` を抽出し、各項目に対応する回答を末尾に追記する
- **各項目の冒頭に必ず「**定義**：〇〇とは、〜である」を置く**（ユーザー要請：口頭で「Xとは？」と聞かれてすぐ答えられるように）
- 提唱者・出典（Freud, Beck, DSM-5, 児童虐待防止法など）も併記して定義の根拠を示す
- 課題シートの時間表示（30秒/1分/2分）は口頭説明の深さの目安。回答の粒度をそれに合わせる
- データ・事例検討（「日本の年間出生数」「米国医療制度比較」「事例の脚色」など答えがそもそもデータ／記述であるもの）は定義不要

### 同一資料の同期が必要なペア
- `5:7` と `5:22` は同じ `課題シート Ch5 8ー…numata.docx` を扱う。両フォルダで **`予習ノート_Ch05_08_行動の評価.md`、`レビューテスト_Ch05_症状評価.md`、`レビューテスト_Ch08_研究手法.md`** を共有する。片方編集したら `cp` で他方へ同期

### レビューテストmdの構造規約
- **1ファイル＝1章**。多章混在ファイル（`Ch02_03`, `Ch05_08`, `Ch06_22`, `Ch14_15`, `Ch18_19`, `Ch20_21`, `Ch23_24` 等）は禁止
- ファイル名は `レビューテスト_ChXX_<テーマ>.md`（命名は1章1テーマ、`グループ20` サフィックスは旧形式）
- 問題見出しは `## 問題N（ChX QY）<タイトル>` 形式。**Nは1から連番、Q番号はBRS PDFの番号を昇順に並べる**
- 1問でBRSの複数Qを統合した場合は `（Ch7 Q3, Q4, Q5）` `（Ch13 Q1-3）` のように列挙／範囲表記。網羅判定スクリプトはこの記法を展開する
- 各章はBRS章末Review Testの **全Q を抜けなく** 収録すること
- 問題本文は **BRSの英文をそのまま転記**＋選択肢全部、解説は和文（提出スライドと同じ流儀）
- 各問末尾に `**【担当グループ】** グループ20　（BRS 第8版 ChX）` を付ける（既存規約。担当外章でも揃える）

### 提出用スライド原稿のテンプレート要件
スライド原稿 .md は次の3ブロック構成を厳守：
1. **■ スライド作成時の注意（テンプレート準拠）** … 4:3固定、選択肢省略不可、和文解説、文末に `【担当グループ】グループ20 ／ 教科書 第8版 ChapterN`
2. **■ スライド1（問題スライド）にそのまま貼る文章** … BRSの英文をそのまま転記＋選択肢全部
3. **■ スライド2（解答解説スライド）にそのまま貼る文章** … 正答＋詳細解説
4. （任意）**■ 発表時の口頭補足メモ** … 2分用ポイント

問題番号変更の依頼があったら、上記構造を保ったまま BRS Chapter X の該当 Q を全部入れ替える。

## よく使う操作

### .docx から本文抽出（pandocなし環境）
```bash
python3 << 'EOF'
import zipfile, xml.etree.ElementTree as ET
NS = "{http://schemas.openxmlformats.org/wordprocessingml/2006/main}"
with zipfile.ZipFile("path/to/課題シート.docx") as z:
    root = ET.fromstring(z.read("word/document.xml").decode("utf-8"))
for p in root.iter(NS+"p"):
    print("".join(t.text or "" for t in p.iter(NS+"t")))
EOF
```

### BRS教科書 PDF からテキスト抽出（章末問題確認用）
```bash
pdftotext -layout "BRS Behavioral Science (8e).pdf" /tmp/brs.txt
# Chapter X の Review Test を探す：grep -n "Review Test" /tmp/brs.txt
# 解答解説は "Answers and Explanations" セクションにまとまっている
```

### レビューテストmdのQ網羅チェック
- 各 `レビューテスト_ChXX_*.md` に対し `## 問題N（ChX Q…）` の Q 番号を抽出し、BRS章末Review Testの最大Qと比較
- `Q1-3` `Q1, 2, 3` の範囲表記も展開し、BRS全Qが揃っているか確認するのが基本動作
- 章末問題数の目安（BRS第8版 / pdftotext抽出ベース）：Ch1=24, Ch2=19, Ch3=15, Ch4=27, Ch5=12, Ch6=22, Ch7=21, Ch8=12, Ch12=20, Ch13=27, Ch14=28, Ch15=15, Ch18=14, Ch19=26, Ch20=15, Ch21=32, Ch22=11, Ch23=42, Ch24=23

## 注意

- 課題シート（.docx）は教員の一次資料。**改変・再保存しない**（`~$…docx` のロックファイルが見えたら開いている可能性があるので触らない）
- 予習ノートに新しい知識を追記する場合、既存の章別解説と「学習項目への回答」の両方に整合する形で書く
- 提出スライド原稿のフッター `【担当グループ】グループ20 ／ 教科書 第8版 ChapterN` は提出要件なので削らない
